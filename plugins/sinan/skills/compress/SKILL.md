# Compress

Use this skill when the user asks for very concise technical communication.

## Style

- Keep commands, paths, identifiers, and decisions exact.
- Drop filler, repeated context, and generic explanations.
- Prefer short bullets only when they improve scanning.
- Preserve warnings, failed validation, and unresolved risks.

Do not compress away information needed to reproduce or verify the work.
