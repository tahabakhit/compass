# Zoom Out

Use this skill when local details are confusing because the surrounding system is not yet mapped.

## Workflow

1. Identify the entry points, owners, and data boundaries.
2. Read the smallest set of files that explains the behavior.
3. Name the existing patterns before proposing changes.
4. Separate stable facts from hypotheses.
5. Return a concise map of components, responsibilities, and likely next files.

For architecture-wide work, pair this with the `architecture-sweep` workflow.
