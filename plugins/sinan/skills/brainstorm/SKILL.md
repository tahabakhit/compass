# Brainstorm

Use this skill when the next useful move is to shape an idea before planning, decision capture, or implementation.

## Workflow

1. Inspect repo docs and code first when the answer may already exist.
2. Identify the highest-leverage unknown blocking progress.
3. Ask one question at a time, with a recommended answer or default when useful.
4. Prefer concrete tradeoffs, non-goals, first users, constraints, and first vertical slice.
5. Track decisions that become stable, but do not write them until decision capture is requested or confirmed.
6. Stop once acceptance criteria, risks, open questions, and the next concrete step are clear.

## Output

End with the explored options, recommended direction, accepted constraints, candidate decisions to capture, and the smallest next step.
